# PRD.md — Product Requirements Document (เอกสารข้อกำหนดผลิตภัณฑ์)

## BUMBIMBANANA Fashion Product Radar

เวอร์ชัน: 1.1 (รองรับข้อมูลจริงจาก Windsor.ai)
สถานะ: รองรับทั้ง Mock Data (ข้อมูลตัวอย่าง) และข้อมูลจริงจาก TikTok Shop ผ่าน Windsor.ai สลับได้ด้วยตัวแปรสภาพแวดล้อม `DATA_SOURCE_MODE`

---

## 1. ภาพรวมและเป้าหมาย

BUMBIMBANANA Fashion Product Radar คือเว็บไซต์ที่ช่วยให้เจ้าของเว็บ (ผู้ใช้คนเดียวในเวอร์ชันนี้) ดู ค้นหา เปรียบเทียบ และคัดเลือกสินค้าแฟชั่นผู้หญิงจาก TikTok Shop ที่มีแนวโน้มขายดี เพื่อนำไปวางแผนทำคอนเทนต์ (Content — เนื้อหา) บน TikTok

เป้าหมายหลักของผู้ใช้คือตอบคำถาม:

* สินค้าไหนยอดขายดี
* สินค้าไหนค่าคอมมิชชันสูง
* สินค้าไหนกำลังเติบโตเร็ว
* สินค้าไหนอยู่ในอันดับต้น ๆ
* สินค้าไหนควรบันทึกไว้ทำคอนเทนต์ภายหลัง

เวอร์ชันแรกนี้**ไม่มีระบบสมัครสมาชิกหรือผู้ใช้หลายคน** รองรับทั้ง **Mock Data** และ**ข้อมูลจริงจาก TikTok Shop ผ่าน Windsor.ai** (ดูหัวข้อ 6)

## 2. กฎสำคัญที่ผูกกับทุก Feature

1. ภาษาไทยทั้งเว็บไซต์ คำอังกฤษต้องมีคำอธิบายไทยในวงเล็บ (ยกเว้นชื่อแบรนด์/เทคโนโลยี)
2. ออกแบบ Mobile First ก่อนค่อยขยายไปคอมพิวเตอร์
3. รองรับทั้ง Mock Data และข้อมูลจริง — ห้าม Scrape ข้อมูลจากแพลตฟอร์มใด ๆ โดยไม่ได้รับอนุญาตเด็ดขาด ข้อมูลจริงต้องมาจากการเชื่อมต่อที่ได้รับอนุญาตอย่างถูกต้องเท่านั้น (เช่น OAuth ผ่าน Windsor.ai) ห้าม Bypass ระบบล็อกอิน/CAPTCHA
4. ห้ามเก็บ/แสดง API Key บนหน้าเว็บไซต์ ต้องอยู่ใน environment variable ฝั่งเซิร์ฟเวอร์เท่านั้น
5. ใช้ Repository Pattern เพื่อสลับแหล่งข้อมูลระหว่าง Mock Data กับ Windsor.ai ได้โดยไม่ต้องแก้โค้ดหน้าเว็บไซต์
6. ทุกหน้าที่เกี่ยวข้องกับข้อมูลสินค้าต้องแสดงสถานะ "ข้อมูลล่าสุด" และโหมดข้อมูลปัจจุบัน (Mock Data หรือ Windsor.ai) ชัดเจน
7. เงินทั้งหมดแสดงเป็นบาท (฿)
8. ตัวอย่างสินค้าทั้งหมดเป็นสินค้าแฟชั่นผู้หญิงเป็นหลัก

## 3. ผู้ใช้งานและลำดับการใช้งาน (User Flow)

ผู้ใช้: เจ้าของเว็บไซต์ (1 คน ไม่มี role อื่น)

ลำดับการใช้งานที่คาดหวัง:

1. เข้าเว็บไซต์ → เห็นหน้า Dashboard (หน้าสรุปภาพรวม) พร้อมสินค้าขายดีทันที
2. เลือกเป้าหมาย: ค่าคอมสูง / ยอดขายแรง / เติบโตเร็ว / รายได้สูง / คะแนนความน่าสนใจสูง
3. เลือกช่วงเวลา: 7 วัน หรือ 30 วัน
4. ไปหน้า Product Radar (หน้าค้นหาและคัดสินค้า) เพื่อดูอันดับสินค้าและกรองข้อมูล
5. เปิดดู Product Detail (หน้ารายละเอียดสินค้า) ของสินค้าที่สนใจ
6. บันทึกสินค้าไว้ในรายการที่สนใจ พร้อมเพิ่มโน้ตส่วนตัว
7. กลับมาดูใน Saved Products (หน้าสินค้าที่บันทึกไว้) ได้ทุกเมื่อ
8. ตรวจสอบความน่าเชื่อถือของข้อมูลผ่านหน้า Data Status (หน้าสถานะข้อมูล)

## 4. ขอบเขตหน้าเว็บไซต์ (Pages Scope)

### 4.1 Dashboard (หน้าสรุปภาพรวม) — `/`

**วัตถุประสงค์**: ให้ภาพรวมตลาดสินค้าแฟชั่นใน 1 หน้าจอ ช่วยตัดสินใจเร็ว

**ต้องแสดง**
- จำนวนสินค้าแฟชั่นทั้งหมดในระบบ
- ยอดขายรวม 7 วัน
- ยอดขายรวม 30 วัน
- ค่าคอมมิชชันรวมโดยประมาณ
- สินค้าขายดีที่สุด (อย่างน้อย 1 รายการเด่น)
- สินค้าที่กำลังโตเร็ว (อย่างน้อย 1 รายการเด่น)
- หมวดสินค้าที่น่าสนใจ
- วันที่อัปเดตข้อมูลล่าสุด
- สถานะ Mock Data (ข้อมูลตัวอย่าง) ชัดเจน

**การควบคุม (Controls)**
- ปุ่มเลือกเป้าหมาย: ค่าคอมสูง / ยอดขายแรง / รายได้สูง / เติบโตเร็ว / คะแนนความน่าสนใจสูง
- ปุ่มเลือกช่วงเวลา: 7 วัน / 30 วัน

**กราฟ (อย่างน้อย 2 แบบ ใช้ Recharts)**
- กราฟยอดขายตามหมวดสินค้า (Bar Chart)
- กราฟเปรียบเทียบยอดขาย 7 วัน vs 30 วัน

### 4.2 Product Radar (หน้าค้นหาและคัดสินค้า) — `/products`

**การแสดงผล**
- คอมพิวเตอร์: ตาราง (TanStack Table) เรียงคอลัมน์ตามข้อมูลสินค้าทั้งหมด
- มือถือ: การ์ดสินค้า (Product Card) แสดงข้อมูลสำคัญ

**ข้อมูลที่ต้องแสดงต่อสินค้า**: รูป, ชื่อสินค้า, หมวดสินค้า, ชื่อร้าน, ราคา, ค่าคอมมิชชัน, ยอดขาย 7 วัน, ยอดขาย 30 วัน, รายได้โดยประมาณ, อัตราการเติบโต, คะแนนความน่าสนใจ, อันดับยอดขาย, อันดับค่าคอมมิชชัน, อันดับการเติบโต, วันที่อัปเดตล่าสุด

**ตัวกรอง**: หมวดสินค้า, ช่วงราคา, ค่าคอมมิชชันขั้นต่ำ, ยอดขายขั้นต่ำ, ช่วงเวลา (7/30 วัน), อัตราการเติบโต, สินค้าที่บันทึกแล้ว

**ตัวเรียงข้อมูล**: ยอดขายสูงสุด, รายได้สูงสุด, ค่าคอมมิชชันสูงสุด, เติบโตเร็วที่สุด, คะแนนความน่าสนใจสูงสุด, อันดับยอดขายดีที่สุด

**ปุ่มต่อรายการ**: ดูรายละเอียด, บันทึกสินค้า / ยกเลิกการบันทึกสินค้า (toggle ตามสถานะ)

### 4.3 Product Detail (หน้ารายละเอียดสินค้า) — `/products/[id]`

**ต้องแสดง**: รูป, ชื่อสินค้า, หมวดสินค้า, ร้านค้า, ราคา, ค่าคอมมิชชัน, ลิงก์สินค้า (เปิดแท็บใหม่), ยอดขาย 7/30 วัน, รายได้โดยประมาณ, อัตราการเติบโต, คะแนนความน่าสนใจ, อันดับยอดขาย/ค่าคอมมิชชัน/การเติบโต, วันที่อัปเดตล่าสุด, ปุ่มบันทึกสินค้า, ช่องเพิ่มโน้ตส่วนตัว

**กราฟ**: เปรียบเทียบยอดขาย 7 วัน vs 30 วัน (รายสินค้า)

**การ์ดสรุปเชิงคุณภาพ**: จุดเด่นของสินค้า, เหตุผลที่น่าสนใจ, เหมาะกับการทำคอนเทนต์แบบไหน (สร้างจาก mock data เชิงบรรยาย)

### 4.4 Saved Products (หน้าสินค้าที่บันทึกไว้) — `/saved`

**ต้องแสดง**: รายการสินค้าที่บันทึก, รูป, ชื่อสินค้า, หมวดสินค้า, ค่าคอมมิชชัน, ยอดขาย, โน้ตส่วนตัว, วันที่บันทึก, ปุ่มเปิดดูรายละเอียด, ปุ่มลบออกจากรายการ

**ตัวกรอง**: หมวดสินค้า, วันที่บันทึก, ค่าคอมสูง, ขายดี

ข้อมูลสินค้าที่บันทึกไว้เก็บใน local storage (ที่จัดเก็บข้อมูลในเบราว์เซอร์) ของเวอร์ชันนี้ เพื่อให้ใช้งานได้จริงโดยไม่ต้องเชื่อมฐานข้อมูลจริงก่อน แต่โครงสร้างพร้อมย้ายไปใช้ตาราง `saved_products` ใน Supabase ได้ทันทีในอนาคต

### 4.5 Data Status (หน้าสถานะข้อมูล) — `/data-status`

**ต้องแสดง**: โหมดข้อมูลปัจจุบัน (Mock Data / Windsor.ai ในอนาคต), วันที่อัปเดตข้อมูลล่าสุด, จำนวนสินค้าในระบบ, สถานะการซิงก์ข้อมูล (Sync Status), พื้นที่เตรียมไว้สำหรับเชื่อม Windsor.ai, คำอธิบายว่าเวอร์ชันนี้ยังใช้ Mock Data

## 5. โครงสร้างข้อมูล (Data Model)

ดูรายละเอียดคอลัมน์ทั้งหมดในตาราง `products`, `saved_products`, `data_sync_logs` ที่ไฟล์ SQL migration `supabase/migrations/`. ระดับ TypeScript กำหนดไว้ใน `types/product.ts`

## 6. โครงสร้าง Data Source / สถาปัตยกรรมการสลับแหล่งข้อมูล

ใช้ Repository Pattern แยกชั้นข้อมูลออกจาก UI:

```
lib/data-source/
  product-repository.ts          ← จุดเดียวที่ UI เรียกใช้ เลือก implementation ตาม DATA_SOURCE_MODE
  mock-product-repository.ts     ← ใช้งานเมื่อ DATA_SOURCE_MODE=mock (ค่าเริ่มต้น)
  windsor-product-repository.ts  ← ใช้งานเมื่อ DATA_SOURCE_MODE=windsor ดึงข้อมูลจริงจาก TikTok Shop ผ่าน Windsor.ai
```

การสลับแหล่งข้อมูลทำผ่านตัวแปรสภาพแวดล้อม `DATA_SOURCE_MODE` (ดู `.env.example`) โดยไม่ต้องแก้โค้ดหน้าเว็บ ฟิลด์บางส่วนจาก Windsor.ai (ค่าคอมมิชชัน, หมวดสินค้า, รูปสินค้า, ลิงก์สินค้า) ยังอยู่ระหว่างปรับให้แม่นยำขึ้นตามชื่อฟิลด์จริงที่ยืนยันได้จากบัญชีที่เชื่อมต่อ

## 7. Non-goals (สิ่งที่ไม่ทำในเวอร์ชันนี้)

* ไม่มีระบบสมัครสมาชิก/ล็อกอิน/หลายผู้ใช้
* ไม่ Scrape ข้อมูลจากเว็บไซต์ใด ๆ โดยไม่ได้รับอนุญาต (ข้อมูลจริงต้องมาจากการเชื่อมต่อที่ได้รับอนุญาตผ่าน OAuth เท่านั้น)
* ไม่มีการเขียนฐานข้อมูลจริงผ่าน Supabase ในเวอร์ชันนี้ (เตรียมแค่ schema/SQL ไว้)

## 8. เกณฑ์ความสำเร็จ (Acceptance Criteria)

* เปิดเว็บไซต์แล้วใช้งานได้ครบ 5 หน้าตามสเปกบนทั้งมือถือและคอมพิวเตอร์
* Mock data สินค้าแฟชั่นผู้หญิงอย่างน้อย 30 รายการ ครอบคลุมหมวดสินค้าที่กำหนดครบ
* ไม่มีข้อความภาษาอังกฤษหลุดบนหน้าเว็บไซต์โดยไม่มีคำอธิบายไทยในวงเล็บ (ยกเว้นชื่อแบรนด์/เทคโนโลยี)
* ทุกหน้าข้อมูลสินค้าแสดงสถานะ "ข้อมูลล่าสุด" และโหมดข้อมูลปัจจุบันชัดเจน
* โครงสร้างโค้ดสลับระหว่าง Mock Data กับ Windsor.ai ได้ด้วยตัวแปรสภาพแวดล้อมเดียว ไม่ต้องแก้โค้ดหน้าเว็บไซต์
