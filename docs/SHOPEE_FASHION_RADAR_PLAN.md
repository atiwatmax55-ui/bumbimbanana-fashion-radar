## สิ่งที่ Feed ยังไม่มี

### ❌ ค่าคอมมิชชัน (Commission Rate)

Feed นี้ไม่มีคอลัมน์ commission rate เลย — ข้อมูล commission ของสินค้า Shopee Affiliate ต้องหาจาก:

- **Shopee Affiliate Center** → ค้นสินค้าใน "Find Products" → ดู commission % รายสินค้า (ทำด้วยมือ)
- **Shopee Affiliate API** (ต้องสมัคร API access แยก ติดต่อ Shopee Partner)
- **Shopee Datafeed ชนิดอื่น** — บาง feed type อาจมีคอลัมน์ commission

> **กฎ:** ห้ามสร้างหรือเดาค่าคอมมิชชัน ระบบจะแสดง "ยังไม่มีข้อมูลค่าคอมมิชชันจาก Feed" เสมอ

## ข้อควรระวัง: ความหมายของ `item_sold`

`item_sold` ใน Shopee Product Feed คือ **ยอดขายสะสมตลอดอายุสินค้า** ไม่ใช่ยอดขายรายวันหรือรายสัปดาห์

- ❌ ห้ามเรียกว่า "ยอดขาย 7 วัน", "ยอดขายวันนี้", หรือ "ยอดขายรายเดือน"
- ✅ เรียกว่า **"จำนวนขายจาก Feed"** หรือ **"ยอดขายสะสม (จาก Shopee)"**
- สินค้าที่เปิดมาหลายปีจะมี `item_sold` สูงกว่าสินค้าใหม่แม้จะขายดีกว่ารายวัน

## ข้อควรระวัง: หน่วยราคา

ราคาใน `sale_price` และ `price` อาจอยู่ในหน่วยที่ต่างจากบาทปกติ:

| หน่วยที่เป็นไปได้ | ตัวอย่าง ฿199 | วิธียืนยัน |
|-----------------|-------------|----------|
| บาท (×1) | `199` | ค่าตรงไปตรงมา |
| สตางค์ (×100) | `19900` | หาร 100 = บาท |
| หน่วย Shopee (×100000) | `19900000` | หาร 100000 = บาท |

**วิธียืนยัน:** เปิด Shopee → เข้าสินค้าจากตัวอย่างใน Feed → เปรียบค่า `sale_price` กับราคาบนหน้าเว็บ

## แผนขั้นตอน (Roadmap)

### ขั้นที่ 1 — ยืนยันหมวดแฟชั่น (ทำได้เลย)

1. กด "วิเคราะห์หมวดแฟชั่นและสินค้าตัวอย่าง" ในหน้า Data Status
2. ดูรายการ `global_category1` ทั้งหมดที่พบ
3. ระบุด้วยมือว่าหมวดไหนคือแฟชั่นสตรีจริง ๆ
4. แจ้งชื่อหมวดที่ยืนยันให้ developer เพื่อ hardcode ใน filter

### ขั้นที่ 2 — ยืนยันหน่วยราคา (ทำได้เลย)

1. ดูค่า Raw จากตาราง "ตัวอย่างค่า sale_price จาก Feed"
2. เปรียบค่ากับราคาจริงบนหน้าสินค้า Shopee
3. แจ้ง developer ว่าต้องหารด้วยเท่าไร (1, 100 หรือ 100,000)

### ขั้นที่ 3 — Import เฉพาะ Top Fashion Products (หลังยืนยัน 1+2)

1. สร้าง `shopee-product-repository.ts` ใน `lib/data-source/`
2. Filter เฉพาะสินค้าที่อยู่ในหมวดแฟชั่นที่ยืนยันแล้ว
3. Map คอลัมน์ตาม `docs/SHOPEE_FEED_COLUMNS.md`
4. แปลงราคาให้ถูกหน่วย (หลังยืนยัน)
5. Import ไม่เกิน 500 รายการแรกก่อน แล้วตรวจความถูกต้อง
6. หาก OK → Import เต็มจำนวน (แนะนำ: เฉพาะสินค้าที่ item_sold > 0)
7. บันทึกลง Supabase ตาม schema ที่เตรียมไว้

### ขั้นที่ 4 — Daily Snapshot เพื่อวัด Trend (ระยะกลาง)

เป้าหมาย: รู้ว่าสินค้าขายดีขึ้นหรือแย่ลงในแต่ละวัน

```
สิ่งที่ต้องทำ:
1. เพิ่ม table ใน Supabase:
   shopee_item_sold_snapshots (
     itemid TEXT,
     snapshot_date DATE,
     item_sold INT,
     PRIMARY KEY (itemid, snapshot_date)
   )

2. ทำ Daily Job (Vercel Cron หรือ GitHub Actions):
   - ดึง Feed ทุกวันตอนตี 3
   - เปรียบ item_sold วันนี้กับเมื่อวาน
   - delta = item_sold_today - item_sold_yesterday
   - delta คือ "ยอดขายในช่วง 24 ชั่วโมง" ที่ประมาณได้

3. ใน Product Radar:
   - แสดง "ยอดขาย 24 ชม. (โดยประมาณ)" จาก delta snapshot
   - ยังคงแสดง "จำนวนขายจาก Feed" สำหรับยอดสะสม
```

> หมายเหตุ: delta จะไม่ 100% แม่นยำ เพราะ Shopee อาจอัปเดต item_sold ไม่ real-time แต่ถือว่าดีกว่าไม่มีข้อมูล trend เลย

### ขั้นที่ 5 — หาแหล่งข้อมูลค่าคอมมิชชัน (ระยะยาว)

ตัวเลือกที่เป็นไปได้:

| แหล่ง | ราคา | ความยาก | ความแม่นยำ |
|-------|------|---------|-----------|
| Shopee Affiliate Center (กรอกมือ) | ฟรี | ง่าย | ✅ 100% |
| Shopee Affiliate API | ต้องสมัคร | ปานกลาง | ✅ 100% |
| Manual CSV Export จาก Affiliate Center | ฟรี | ปานกลาง | ✅ 100% |
| Datafeed type อื่นจาก Shopee | ไม่ทราบ | ไม่ทราบ | ไม่ทราบ |

**แนะนำ:** เริ่มจาก Manual Curation (กรอกมือ) สำหรับสินค้าที่สนใจจริง ๆ แล้วสมัคร Shopee Affiliate API ในระยะกลาง
